# Contributing to Star Wars: RebellionHelper

Thank you for your interest in contributing to RebellionHelper! This guide will help you get started.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an [issue](https://github.com/ZiNNED/rebellionhelper.app/issues) and include:

- A clear description of the problem
- Steps to reproduce the issue
- What you expected to happen vs. what actually happened
- Screenshots, if applicable

### Suggesting Features

Feature ideas are welcome! Please open an [issue](https://github.com/ZiNNED/rebellionhelper.app/issues) with the **enhancement** label and describe:

- What the feature would do
- Why it would be useful
- Any implementation ideas you have

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b my-feature`)
3. **Make your changes** and commit them with clear, descriptive messages
4. **Push** your branch to your fork (`git push origin my-feature`)
5. **Open a Pull Request** against the `master` branch

### Pull Request Guidelines

- Keep PRs focused — one feature or fix per PR
- Describe what your changes do and why
- Include screenshots for any visual changes
- Make sure the app still works correctly after your changes
- The repository owner will review all PRs before merging

## Development Setup

This is a static web app. To run it locally:

1. Clone your fork
2. Open `index.html` in a browser, or serve the directory with any static file server:
   ```bash
   # Using Python
   python3 -m http.server 8000

   # Using Node.js (npx)
   npx serve .
   ```

## Code of Conduct

Be respectful and constructive in all interactions. We're all here because we love Star Wars: Rebellion!

## Questions?

If you have questions, feel free to open an issue — we're happy to help.
